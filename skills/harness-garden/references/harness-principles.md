# Harness Principles

`harness-garden` protects the same principles that `harness-bootstrap`
establishes:

- root `AGENTS.md` is a map, not a handbook
- progressive disclosure beats one giant instruction file
- plans, manifests, and generated facts are the continuity layer for long-running work
- observability and runtime feedback surfaces are first-class harness assets
- recurring entropy should be tracked explicitly, not left as folklore
- low-risk drift can be repaired automatically
- high-risk semantic drift should go through an explicit plan

Use the local anchor
`http://refinex.cn/blog/harness-engineering-the-control-plane-for-agent-first-software`
and the OpenAI / Anthropic harness articles as the narrative basis when
deciding whether a drift issue is structural, planning-related,
runtime-visibility-related, or generated-fact related.
