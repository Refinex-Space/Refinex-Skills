# Root AGENTS.md Guide

The root `AGENTS.md` is the primary entry point for any coding agent working in the repository. It is a **table of contents**, not an encyclopedia. Its job is to orient the agent and point it to deeper sources of truth — never to contain all the truth itself.

## Design Principles

1. **~100 lines maximum.** Context is a scarce resource. A giant instructions file crowds out task-specific context and code. When everything is "important", nothing is.
2. **Map, not manual.** Point to docs/, not repeat them. An agent that needs architecture details should follow a link to `docs/ARCHITECTURE.md`, not read about architecture inline.
3. **Grounded in reality.** Every command, path, and constraint in AGENTS.md must reflect the actual repo. If `./gradlew test` doesn't work, don't list it.
4. **Progressive disclosure.** The agent starts with AGENTS.md's ~100 lines, then follows links to go deeper only when needed for a specific task.

## What NOT to Include

- **General programming advice.** "Write clean code" and "follow SOLID principles" waste tokens.
- **Things Claude already knows.** Don't explain what Spring Boot is or how npm works.
- **Information that's one `cat` call away.** If the agent can discover it by reading a config file, don't duplicate it in AGENTS.md.
- **Detailed API documentation.** Link to docs instead.
- **Information that changes frequently.** AGENTS.md should be stable. Volatile information belongs in docs/ or generated/ files.

## Handling Existing AGENTS.md

If the repo already has an AGENTS.md:

1. **Read it completely.**
2. **If it's already a good TOC** (<100 lines, links to docs): add any missing harness sections (Documentation Index, Harness section). Preserve everything else.
3. **If it's a moderate encyclopedia** (100-300 lines): extract the valuable content into appropriate docs/ files, then refactor AGENTS.md into a TOC that links to them. Ask the user first.
4. **If it's a large encyclopedia** (>300 lines): do NOT refactor without user approval. Ask: "The existing AGENTS.md is {N} lines. The harness pattern recommends ~100 lines as a table of contents. Should I refactor it, or leave it and add harness links alongside?"
