# Office Skills 套件说明

## 概述

Office Skills 套件把四个面向文件格式的 Skill 统一归到 `office-*`
命名空间下。把原来的裸扩展名式命名改成带前缀命名，不是样式调整，而是
为了三个明确目标：让提示词路由更清晰、让技能索引更容易扫描、并明确这四个
Skill 属于同一组文档交付能力，而不是彼此孤立的工具说明。

## 四个 Skill 的定位

### 1. `office-docx`

用于 Word 文档的创建、编辑、XML 级修复、修订处理，以及 `.doc` 到
`.docx` 的转换流程。

适合场景：

- 正式报告、备忘录、信函、模板等 `.docx` 交付物
- 修订接受、批注处理、XML 结构修复
- 需要保留分页、标题层级或版式约束的 Word 输出

### 2. `office-pdf`

用于 PDF 的读取、提取、生成、转换、表单填写、OCR 相关流程和图像转换。

适合场景：

- 从 PDF 提取文本或表格
- 填写可填写或不可填写的 PDF 表单
- 拆分、合并、旋转、加水印、加密或生成 PDF

### 3. `office-pptx`

用于幻灯片、Pitch Deck、演示模板和 PowerPoint 编辑流程。

适合场景：

- 创建或修改 `.pptx` 交付物
- 审阅演示文稿的结构与抽取出的内容
- 需要缩略图、解包和回包的模板化幻灯片工作流

### 4. `office-xlsx`

用于电子表格的创建、清洗、分析、格式处理，以及保持公式可用的工作簿交付。

适合场景：

- 最终结果必须保持为电子表格的 `.xlsx`、`.xlsm`、`.csv`、`.tsv` 任务
- 工作簿修复、重算、保持模板风格的更新
- 需要保留动态公式而不是写死结果值的表格交付物

## 经典使用案例

### Word 文档优先的交付物

```text
$office-docx
任务：把这份访谈笔记整理成可审阅的 .docx 报告，包含标题层级、页码和批注
```

### PDF 提取或表单流程

```text
$office-pdf
任务：从这份季度 PDF 提取表格，并填写附带的申请表单
```

### 演示文稿修订

```text
$office-pptx
任务：更新这份 investor deck，保持模板风格，并输出可交付的 .pptx 文件
```

### 以电子表格为最终产物

```text
$office-xlsx
任务：清洗这份 CSV，恢复公式，并交付一个没有公式错误的最终 .xlsx 工作簿
```

## 推荐使用方式

先按最终交付物选 Skill：

- 输出是 Word 文档时，使用 `office-docx`
- 输出是 PDF，或 PDF 是主要事实来源时，使用 `office-pdf`
- 任务中心是幻灯片或演示文件时，使用 `office-pptx`
- 输出必须保持为电子表格工作簿时，使用 `office-xlsx`

## 目录结构

```text
skills/
├── office-docx/
├── office-pdf/
├── office-pptx/
└── office-xlsx/
```

每个 Skill 各自维护 `SKILL.md` 以及该文件格式所需的本地脚本或参考材料。
